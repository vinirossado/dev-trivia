📊 Table View 🔀 ER Diagram

🎯 Dev Trivia System – Entity Relationship Diagram

erDiagram

    Users {
        uuid Id PK
        varchar Name
        varchar Email
        text PasswordHash
        text ProfileImageUrl
        varchar AuthProvider
        varchar ExternalId
        text Bio
        varchar Location
        date DateOfBirth
        timestamp LastLoginAt
        varchar PreferredLanguage
        timestamp CreatedAt
        timestamp UpdatedAt
    }

    AuthSessions {
        uuid Id PK
        uuid UserId FK
        text RefreshTokenHash
        timestamp ExpiresAt
        timestamp RevokedAt
    }

    UserRoles {
        uuid UserId PK
        varchar Role
    }

    PlayerStats {
        uuid UserId PK
        int TotalMatches
        int TotalCorrect
        int EloRating
    }

    QuestionExposure {
        uuid UserId PK
        uuid QuestionId PK
        timestamp LastSeenAt
        int TimesShown
    }

    Categories {
        uuid Id PK
        nvarchar Name
        nvarchar Description
    }

    Questions {
        uuid Id PK
        nvarchar Title
        nvarchar Description
        nvarchar Difficulty
        uuid CategoryId FK
    }

    AnswerOptions {
        uuid Id PK
        uuid QuestionId FK
        nvarchar Text
        boolean IsCorrect
    }

    Matches {
        uuid Id PK
        datetime StartedAt
        datetime EndedAt
        nvarchar Status
        uuid SelectedCategoryId FK
    }

    PlayerAnswers {
        uuid Id PK
        uuid MatchId FK
        uuid UserId FK
        uuid QuestionId FK
        uuid AnswerOptionId FK
        boolean IsCorrect
        datetime AnsweredAt
    }

    Users ||--o{ AuthSessions : has
    Users ||--o{ UserRoles : has
    Users ||--|| PlayerStats : has
    Users ||--o{ QuestionExposure : tracks
    Questions ||--o{ QuestionExposure : exposed_to
    Categories ||--o{ Questions : categorizes
    Questions ||--o{ AnswerOptions : has
    Matches ||--o{ PlayerAnswers : contains
    Users ||--o{ PlayerAnswers : answers
    Questions ||--o{ PlayerAnswers : evaluated_in
    AnswerOptions ||--o{ PlayerAnswers : selected_in
    Categories ||--o{ Matches : selected_for
🔗 Key Relationships: Users → AuthSessions, UserRoles, PlayerStats (User authentication and statistics)
Users ↔ QuestionExposure (Tracks which questions users have seen)
Categories → Questions → AnswerOptions (Questions are categorized and have answer options)
Matches → PlayerAnswers (Matches contain player answers)
PlayerAnswers connects Users, Matches, Questions, and AnswerOptions
